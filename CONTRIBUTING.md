# Contributing to Refactory Apprenticeship Program

Thank you for your interest in contributing to the Refactory Apprenticeship Program! We welcome contributions from students, mentors, and the wider tech community to help improve our learning resources.

---

## 📖 Overview

The **Refactory Apprenticeship** is a six-month intensive program designed to equip aspiring software developers with advanced technical skills and real-world industry experience. Through structured training, project-based learning, and mentorship, apprentices gain the expertise needed to thrive in the tech ecosystem and build impactful digital solutions.

You can find more details about the official program on the [Refactory Apprenticeship website](https://refactory.academy/programs/apprenticeship/).

### 🎯 Program Impact Framework

- **Advanced Principles**: Learn UML system modeling, Object-Oriented Programming (OOP), and scalable system design.
- **Diverse Tech Stack**: Hands-on experience with modern programming languages (JavaScript, Python, Java) and industry-standard frameworks.
- **Real-World Experience**: 17 weeks of collaborative software development building solutions for leading tech corporations and SMEs.
- **Professional Growth**: Develop soft skills, leadership, and a strong professional portfolio.

---

## 📂 Repository Structure & Curriculum

This repository is organized into phases following the 24-week program structure:

### Phase 1: Learning Phase (8 Weeks)
*   **Module 1: UML** - System modeling diagrams.
*   **Module 2: OOP Basics** - Building scalable and maintainable systems.
*   **Module 3: TDD (Current)** - Test-Driven Development fundamentals.
*   *(Additional modules like Web/Mobile frameworks to be added)*

### Phase 2: Project Development Phase (17 Weeks)
*   **Project Initiation**: Mapping and Research.
*   **Collaborative Development**: Building industry-grade projects.

## 🚀 Getting Started

### 1. Set Up Your Environment
Ensure you have the following installed:
- **Python 3.12+**
- **[uv](https://github.com/astral-sh/uv)** (Our preferred package manager)

### 2. Fork and Clone
1. Fork the repository to your own GitHub account.
2. Clone your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/refactory-apprenticeship-program.git
   cd refactory-apprenticeship-program
   ```

### 3. Create a Branch
Always create a new branch for your work:
```bash
git checkout -b feature/your-feature-name
# OR
git checkout -b fix/issue-description
```

## 🏆 Standards & Best Practices

As this program covers various modules from UML to Project Development, we expect all contributions to adhere to industry-standard best practices:

1.  **System Modeling**: Use clear, standard UML notations for design contributions.
2.  **OOP Principles**: Focus on scalability, maintainability, and clean Object-Oriented design.
3.  **Testing**: For code-heavy modules (like TDD or Web Apps), ensure comprehensive test coverage exists.
4.  **Documentation**: Every new exercise or module must include an `INSTRUCTIONS.md` or updated module `README.md`.

---

## 📬 Submitting Changes

1. **Commit your changes**: Use clear, descriptive commit messages.
   ```bash
   git commit -m "feat: add calculator division exercise"
   ```
2. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```
3. **Open a Pull Request (PR)**:
   - Provide a clear description of your changes.
   - Reference any related issues (e.g., `Fixes #123`).
   - Wait for a mentor or maintainer to review your PR.

As part of the Refactory Academy community, we expect all contributors to be respectful, inclusive, and professional. We are all here to learn and grow together!
