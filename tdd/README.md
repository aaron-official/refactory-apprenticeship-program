# TDD Apprenticeship Module

This repository contains exercises and examples for a **Test‑Driven Development (TDD)** training module taught by *Samson Kapeyi*. Each day focuses on a different aspect of TDD, building skills from basic testing through full application development.

## Course Outline

| Day | Topics                        | Activities                                                                 | Outcome                                      |
|-----|-----------------------------|---------------------------------------------------------------------------|----------------------------------------------|
| 1   | Intro to TDD & Why It Matters | Lecture on TDD principles<br>Demo: Red‑Green‑Refactor on a simple function | Understand the TDD mindset                   |
| 2   | Unit Testing Fundamentals     | Explore frameworks (pytest, etc.)<br>Write first test suite               | Comfort with testing tools                   |
| 3   | First Full TDD Cycle          | Build a small module (e.g. Calculator) using TDD                         | Experience iterative development            |
| 4   | Refactoring & Design via TDD  | Discuss code smells and patterns<br>Refactor with confidence using tests | Appreciate clean code through tests         |
| 5   | Working with Dependencies     | Learn mocks, stubs, fakes<br>Apply to functions with I/O or API calls    | Handle complex test scenarios               |
| 6   | TDD for Web or CLI Apps       | Build a small app (to‑do list, note app) via TDD                         | Integrate TDD into app development          |
| 7   | Testing Edge Cases & Errors   | Property‑based testing, boundary analysis                                | Broaden test coverage                       |
| 8   | Legacy Code & Refactoring     | Add tests to untested codebase safely                                   | Practice test‑before‑refactor               |
| 9   | Team TDD & Pair Programming    | Pair TDD exercises, code reviews, CI discussion                          | Collaboration & workflow                    |
| 10  | Capstone Project & Review      | Small group project applying full TDD cycle<br>Retrospective             | Confidence using TDD in real projects       |

---

### How to Use

- Each day lives in its own `day_N/` folder containing instructions, code, and tests.
- Activate the virtual environment and run pytest via `uv run pytest` or similar.
- Follow the `INSTRUCTIONS.md` files to work through examples.

### Goals

- Teach a disciplined approach to writing tests first
- Build habits of small, incremental development
- Empower developers to write maintainable, well‑tested code

Feel free to explore, modify, and extend the exercises as you learn the TDD workflow!
